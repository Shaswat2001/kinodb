1. #[derive()]
    #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
    pub struct EpisodeId(pub u64);

    It automatically generates implementations of common traits for the structure so that we dont have to write them 
    automatically. 

    -> Debug - Allows struct to be printed for debugging
    -> Clone - Allows to explicitly duplicate the value using .clone()
    -> Copy - Allows type to be copied implicitly instead of moved.
    -> PartialEq - Allows for comparison using == and !=
    -> Eq - Indicates full equality semantics
    -> Hash - Allows struct to be used as key in hash based collections. 

2. let file_size = std::fs::metadata(path)?.len();
    std::fs::metadata(path) returns Result<Metadata> which is Ok(Metadata) if the file exists and Err(error) if there is an error.
    ? after that checks if there is an error, throw that if not go to the next bit and calculate the length using .len()

    Its equivalent to - 
        let metadata = match std::fs::metadata(path) {
            Ok(m) => m,
            Err(e) => Return Err(e),
        }

3. for i in 0..reader.num_episodes() 
    Looping within the desired range, meaning looping from 0 but not including reader.num_episodes()
    so if reader.num_episodes() is equal to 5, the loop will run from 0, 1, 2, 3, 4

4. Enum creation - 
    pub enum MixError {
        Read(ReadError),
        NoSources,
        ZeroWeight { path: String },
        Io(std::io::Error),
    }

    Read -> Error caused by reading something. Tuple variant.
    NoSources -> Unit variant 
    ZeroWeight -> Struct error with no weight, path is which struct path it is.
    Io -> Wraps standard IO errors.

5. Custom error handling for Enum - 
    impl std::fmt::Display for MixError {
        fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
            match self {
                MixError::Read(e) => write!(f, "read error: {}", e),
                MixError::NoSources => write!(f, "mixture has no sources"),
                MixError::ZeroWeight { path } => write!(f, "zero weight for {}", path),
                MixError::Io(e) => write!(f, "I/O error: {}", e),
            }
        }
    }

    This defines:
        How MixError should be printed as a user-friendly message.
        Without this, printing would look like:
            Read(ReadError(...))
        With this, you get:
            read error: failed to parse dataset

    impl std::error::Error for MixError {}
